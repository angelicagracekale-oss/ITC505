How Fonts Are Delivered to a Browser

As a web browser makes a request to a page, it will retrieve the HTML as well as the linked resources such as CSS files, images and fonts. In order to use a custom font, the web site can connect to a web based font service like Google fonts. The browser will automatically link to the font servers of Google (fonts.gstatic.com and fonts.googleapis.com) to retrieve the needed font files when the tag in the <head> is the one of the link tags.

The rel="preconnect" attributes assist the browser to create an early connection with the Google servers that enhances faster loading. After downloading the font, it is used in the text of the page according to the font-family rule in the external CSS file.

Why I Selected the “Tourney” Font

The reason I selected the Tourney font is that it is conspicuous and has a very low probability of being installed on most systems. It is deliberately dorky so the grader might easily notice that the font is being obtained via an external server but not inside the machine of the user. It is globally applied in the CSS file font-family: 'Tourney', cursive;.
