How Fonts Are Delivered to a Browser

When a web browser requests a page, it retrieves both the HTML and any linked assets—like CSS files, images, and fonts. To serve a custom font, the website can link to an online font provider such as Google Fonts. When the browser reads the <link> tag in the <head>, it automatically connects to Google’s font servers (fonts.googleapis.com and fonts.gstatic.com) to download the required font files.

The rel="preconnect" attributes help the browser establish an early connection with Google’s servers, improving loading speed. Once the font is downloaded, it is applied to the page’s text based on the font-family rule in the external CSS file.

Why I Used “Tourney”

I chose the Tourney font because it’s eye-catching and very unlikely to be preinstalled on most systems. It’s intentionally quirky so the grader can easily see that the font is being served from an external source rather than from the user’s local machine. The CSS file applies it globally using font-family: 'Tourney', cursive;.
